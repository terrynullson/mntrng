# День, если что (@shoza_den)

Бот для Telegram-канала: напоминает, какой сегодня день недели, с разной степенью абсурда и тона (лениво, эмоционально, с лёгким безумием). Без личного бренда и глубоких смыслов.

## Архитектура

- **Логика публикаций** — по расписанию (утром). Этап 1: автоматическая отправка из заготовок. Этап 2: 5 вариантов от ИИ → выбор/редакт в личку → тайм-аут с «ворчливой» заглушкой при отсутствии ответа.
- **Модульность ИИ** — один интерфейс, смена провайдера в настройках без правок кода:
  - `ai/simple` — заготовленные фразы (этап 1).
  - `ai/yandex` — Яндекс GPT (заглушка под реализацию).
  - `ai/deepseek` — DeepSeek (заглушка).
  - `ai/local` — локальная модель, например Ollama (заглушка).
- **База** — SQLite, таблица постов с полем `approval_status`: `approved_manual`, `timeout`, `manual_full`, `rejected` — для аналитики и статистики.

## Этапы

| Этап | Что сделано |
|------|-------------|
| **1** | По расписанию один пост из списка заготовок, запись в БД. Запуск «один раз» для проверки: `python run.py once`. |
| **2** | Подключение ИИ через фабрику, 5 вариантов в личку, кнопки выбора/редакта, тайм-аут с заглушкой. |

## Запуск

1. Создать бота в @BotFather, добавить бота в канал как админа с правом постить.
2. Скопировать `.env.example` в `.env`, заполнить:
   - `SHOZA_BOT_TOKEN` — токен бота
   - `SHOZA_CHANNEL_ID` — @shoza_den или `-100xxxxxxxxxx`
   - при необходимости `SHOZA_ADMIN_CHAT_ID` (для этапа 2)
3. Установить зависимости: `pip install -r requirements.txt`

### Как именно он будет запускаться

**Вариант А — долгоживущий процесс (сейчас так и задумано)**  
Запускаешь один раз и не выключаешь (на VPS, домашнем ПК, в screen/tmux):

```bash
cd telegram-day-bot
python run.py
```

Процесс висит в фоне: спит до следующего 7:00 (или до времени из `SHOZA_POST_HOUR`/`SHOZA_POST_MINUTE`), в этот момент отправляет пост и снова засыпает до завтра. Нужно, чтобы машина была включена в момент поста.

**Вариант Б — по расписанию системы (без постоянно работающего процесса)**  
Бот не крутится всё время, а вызывается раз в день по cron (Linux/macOS) или Планировщику заданий (Windows):

- Команда на запуск: **`python run.py once`** — один пост на сегодня (если ещё не постили).
- В cron, например в 7:00 каждый день:
  ```cron
  0 7 * * * cd /путь/к/telegram-day-bot && python run.py once
  ```
- В Windows: Планировщик заданий → новая задача → триггер «ежедневно 7:00» → действие: запуск `powershell.exe` с аргументом `-File run_daily.ps1` (или напрямую `py -3 run.py once`), рабочая папка — `telegram-day-bot`.

Расписание по умолчанию в коде — 7:00 локального времени (меняется через `SHOZA_POST_HOUR`, `SHOZA_POST_MINUTE`); при варианте Б важно, чтобы системное время и это время совпадали по смыслу (один раз в день в 7:00).

## Смена провайдера ИИ

В `.env` выставить:
```env
SHOZA_AI_PROVIDER=simple   # или yandex | deepseek | local
```
Для `yandex` / `deepseek` / `local` дописать соответствующие ключи/URL в `.env` и реализовать методы в папках `ai/yandex`, `ai/deepseek`, `ai/local` (сейчас там заглушки с `NotImplementedError`).

## База данных

Один файл SQLite — по умолчанию `data/posts.db` внутри папки бота. Папку `data/` бот создаёт сам при первом запуске, вручную ничего класть не нужно. В БД — только таблица постов (дата, текст, день недели, статус одобрения). Схема в `db/schema.sql`. Путь можно переопределить через `SHOZA_DATA_DIR` в .env (если не задан — используется `data/` рядом со скриптом).

## Структура проекта

```
telegram-day-bot/
├── ai/
│   ├── interface.py   # единый интерфейс TextGenerator
│   ├── factory.py     # get_generator(provider) → реализация
│   ├── simple/       # заготовки (работает)
│   ├── yandex/       # заглушка под API
│   ├── deepseek/     # заглушка под API
│   └── local/        # заглушка под Ollama/другое
├── db/
│   ├── schema.sql
│   ├── schema.py     # init_db
│   └── repository.py # insert_post, get_post_by_date, stats_by_approval
├── config.py         # настройки из .env
├── run.py            # расписание + отправка поста
├── requirements.txt
├── .env.example
└── README.md
```

Проект в репозитории вынесен в отдельную папку и не затрагивает основной код платформы мониторинга.
