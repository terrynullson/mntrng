# UI Style Guide (Flat, modern, professional)

Цель: единый визуальный язык админки. Flat-дизайн, минимализм, строгая типографика, читаемые таблицы, нейтральные цвета, аккуратные микро-анимации.
Главное: предсказуемость, плотность информации, скорость восприятия статусов.

## 1) Design Principles
- Flat first: без "тяжёлых" градиентов, без стеклянных эффектов, без неоморфизма.
- Акцент на контент: таблицы, фильтры, статусы, метрики.
- Иерархия: заголовок → фильтры → таблица/карточки → детали.
- Consistency: одинаковые отступы, радиусы, размеры.
- Motion: микро-анимации только для обратной связи (hover, open/close, loading), без "шоу".

## 2) Layout Rules
- Использовать 12-колоночную сетку (CSS grid) и контейнеры фиксированной ширины.
- Отступы: 4px scale (4/8/12/16/24/32/48).
- Радиусы: 12px–16px (карты 16px, кнопки 12px).
- Тени: мягкие и редкие. В идеале: 1-2 уровня тени, не более.
- Линии разделения: тонкие (border), вместо визуального "объёма".

## 3) Typography
- Базовый шрифт: system-ui (без кастомов на старте).
- Размеры:
  - Page title: 20–24
  - Section title: 16–18
  - Body: 14–15
  - Table: 13–14
- Высота строки: 1.4–1.6.
- Важное правило: не использовать капслок на кнопках/лейблах.

## 4) Color & Status (semantic)
Цвета — только семантические (текст/бордер/фон), никаких "кислотных".

Статусы:
- OK: зелёный (subtle)
- WARN: янтарный
- FAIL: красный
- INFO: синий/нейтральный

Правила:
- Статус отображать бейджем (badge) + точка/иконка.
- Не заливать весь row ярким цветом. Максимум — лёгкий фон или бордер слева.

## 5) Components (must-have)
- Topbar + Sidebar (иконки + разделы)
- Table:
  - sticky header
  - column resize (опционально)
  - sorting, filtering, search
  - pagination
  - row actions (kebab menu)
- Filters:
  - search input
  - status dropdown
  - date range
- Cards:
  - для summary метрик (OK/WARN/FAIL counts)
- Toast notifications
- Empty states (когда данных нет)
- Skeleton loaders (вместо спиннеров)

## 6) Motion & Animation
Используем Framer Motion для:
- page transitions (легко)
- раскрытие фильтров/панелей
- hover/press у кнопок
- появление toast

Правила:
- Длительности: 120–220ms (micro), 240–360ms (panels)
- Easing: easeOut
- Без bounce/elastic.
- Анимации не должны мешать чтению таблиц.

## 7) Data density (admin-first)
- Таблицы компактные: row height 44–52px.
- Иконки маленькие, текст читабельный.
- Любая страница должна быть полезной без скролла на 1080p.

## 8) Accessibility
- Контраст текста достаточный
- focus ring обязателен
- кликабельные зоны >= 40px

## 9) Forbidden
- Неоморфизм, glassmorphism, heavy gradients
- 5+ разных шрифтов/размеров в одном экране
- Тяжёлые анимации и параллакс
- "Радужные" цвета и чрезмерная декоративность

## 10) Definition of Done (UI)
- Страница имеет: заголовок, фильтры, таблицу, состояния загрузки/пусто/ошибка
- Единые компоненты (badge/button/table)
- Семантические статусы
- Микро-анимации соответствуют правилам
